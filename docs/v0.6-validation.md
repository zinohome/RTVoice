**日期**：2026-04-30

## 设计

v0.6 把 agent-worker 从**自写状态机**迁移到 **livekit-agents** 1.5.6 框架。

**保留 v0.5.1 main.py 不动**，新增并行实现：
- `app/plugins.py` — 自定义 STT/TTS plugin 子类（包装现有 stt_client / tts_client）
- `app/main_v06.py` — `AgentSession` entrypoint
- `Dockerfile` CMD 按 `AGENT_BACKEND` 环境变量分支
  - `AGENT_BACKEND=` 或不设 → 跑 v0.5.1（已验证）
  - `AGENT_BACKEND=v06` → 跑 v0.6（实验性）

## 已知风险点

1. **API 版本敏感**：livekit-agents 1.x API 仍在演进；plugin 抽象基类签名（`_run`, `output_emitter` 等）若变化需要适配
2. **silero VAD 网络依赖**：`silero.VAD.load()` 首次会下载模型；prod 离线网需要预热或本地缓存
3. **EventEmitter / asyncio.Queue 集成**：自定义 RecognizeStream `_run()` 里手动 `self._event_ch.send_nowait(...)`，若 framework 改了内部 channel 协议会出问题
4. **ChunkedStream `output_emitter` API**：v1.5.6 是 `initialize() / push() / flush()`，未来版本可能改
