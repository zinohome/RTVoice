## ⚠️ 性能现实：dev 沙盒 CPU 太老

实测 Kokoro 在 **Xeon E5-2697 v2 (2013)** 上：

| 输入 | TTFB | 总耗时 | 音频长度 | RTF |
|---|---|---|---|---|
| 5 字 | 11s | 11s | 1.26s | 0.11× |
| 24 字 | 23s | 23s | 4.52s | 0.19× |

Voice Agent 实时对话需要 **TTFB < 500ms / RTF > 1.0×**。**本环境不达标**。

**根因**：CPU 无 AVX2 指令集，ONNX Runtime 走 fallback 慢路径。**与方案设计无关**。

**预期表现**（其他硬件）：
- 现代 CPU（Ryzen 3000+ / Intel 8 代+）：RTF 1-2×，可用
- 用户家 RTX 3060：v0.5 prod 切 CosyVoice 2 GPU，RTF 5-10×，丝滑

## ⚠️ 未自动验证（需真音频/真 mic）

- 端到端延迟（用户说话 → 听到回复）
- VAD 触发 → STT 流式 partial 出现节奏
- LLM 真实对话质量（Qwen2.5-1.5B 中文表现）
- TTS 真实听感（Kokoro zf_xiaobei 中文自然度）
- Barge-in：agent 说话期间打断 → TTS HTTP stream 取消是否干净

## 🧑 人工验证步骤（在用户的 home box 上）

1. `git pull` 拿 v0.5
2. `cp .env.example .env` 并填入随机密钥（参考 v0.1-v0.4 已有 .env 可复用）
3. `./scripts/dev-up.sh`（首次会拉 Kokoro 模型 + Qwen2.5-1.5B，~5 分钟）
4. 浏览器 `http://127.0.0.1:8000/`
5. 开麦克风，对它说话
6. 期望：听到 Kokoro 中文女声回复（非 sine wave 蜂鸣）
7. agent 说话期间打断，观察是否快速停止

如果 home box 是现代 CPU，TTFB 应在 1-3 秒内，整段回复合成 < 5s。
